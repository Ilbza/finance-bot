# Сценарий Telegram-бота для учёта финансов

## 1. Цель
Бот фиксирует доходы и расходы, считает баланс за период, контролирует лимиты по категориям и даёт быстрые итоги в чате.

## 2. Сущности и правила

### 2.1 Пользователь
- `user_id` (Telegram ID)
- `currency` (`RUB`/`USD`/`EUR`/другая)
- `default_period` (`month`/`week`)
- `cycle_start_day` (1-31, опционально)
- `limits_enabled` (bool)

### 2.2 Категория
- Предустановленные:
  - `Еда`, `Транспорт`, `Кафе`, `Дом`, `Здоровье`, `Развлечения`, `Подписки`, `Одежда`, `Подарки`, `Другое`
- Пользовательские категории:
  - создаются по подтверждению пользователя
  - должны быть уникальны в рамках пользователя (без учёта регистра)

### 2.3 Транзакция
- `id`
- `type`: `expense` | `income`
- `amount` (> 0)
- `category` (обязательно для расхода, для дохода можно `Зарплата`/`Подработка`/`Возврат`/`Другое`)
- `datetime`
- `comment` (опционально)
- `currency`

### 2.4 Лимит
- `category`
- `period`: месяц
- `amount` (> 0)

### 2.5 Расчёты
- Период по умолчанию: текущий месяц.
- `income_total` = сумма доходов периода.
- `expense_total` = сумма расходов периода.
- `balance` = `income_total - expense_total`.
- По категориям: группировка расходов по `category`.

### 2.6 Статус лимита
- `ok`: < 80%
- `80%`: >= 80% и <= 100%
- `exceeded`: > 100%

При каждом добавлении расхода бот проверяет лимит и сразу отправляет статус.

## 5. Онбординг

### Шаг 1: приветствие и валюта
Бот:
`Привет! Я помогу вести бюджет без табличек 😌`
`Считаем по текущему месяцу. Выбери валюту:`

Кнопки: `₽ RUB` / `€ EUR` / `$ USD` / `Другое`

### Шаг 2: лимиты
Бот:
`Хочешь включить лимиты по категориям?`

Кнопки: `Да, настроить` / `Пока нет`

Если `Да, настроить`:
1. Бот даёт чеклист категорий.
2. По каждой выбранной спрашивает сумму.
3. Сохраняет лимиты.

Финал:
`Супер. Теперь просто пиши: еда 450 или жми кнопки меню.`

## 7. Добавление расхода

### Режим A (кнопки)
1. Пользователь: `➕ Добавить расход`
2. Бот: выбор категории
3. Бот: запрос суммы
4. Бот: запрос комментария (`Пропустить`)
5. Бот: подтверждение + баланс + статус лимита

Шаблон подтверждения:
`Записал(а): {category} — {amount} {currency} ✅`
`{category}: {spent}/{limit} ({percent}%) {status}`
`Баланс месяца: {balance}`

### Режим B (текст)
Парсер распознаёт:
- `<категория> <сумма>`
- `<сумма> <категория>`

Если категория неизвестна:
`Категория «{name}» новая. Создать?`
Кнопки: `Да` / `Нет, выбрать из списка`

## 8. Добавление дохода

### Через меню
1. Пользователь: `➕ Добавить доход (зарплата)`
2. Бот: тип дохода (`Зарплата`, `Подработка`, `Возврат`, `Другое`)
3. Бот: сумма
4. Бот: комментарий (опционально)
5. Бот: подтверждение

### Быстро
- `зп 250000`
- `доход 50000`

Шаблон:
`Доход добавлен: {income_type} +{amount} {currency} ✅`
`Баланс месяца: {balance}`

## 9. Итоги

Команда/кнопка: `📊 Итоги`

Ответ:
`📊 Итоги за {month} {year}`
`Доходы: {income_total} {currency}`
`Расходы: {expense_total} {currency}`
`Баланс: {balance} {currency}`
`Топ категорий:`
`1) {cat1} — {sum1}`
`2) {cat2} — {sum2}`
`3) {cat3} — {sum3}`

Кнопки:
- `По категориям`
- `По дням`
- `Экспорт (CSV)`
- `Поставить лимиты`

## 10. Лимиты

Экран:
`Лимиты на этот месяц:`
`Еда: 30 000 (108%) 🚨`
`Дом: 25 000 (84%) ⚠️`
`Кафе: 6 000 (76%) ✅`

Кнопки:
- `➕ Установить лимит`
- `✏️ Изменить лимит`
- `🧹 Сбросить лимиты`
- `⛔️ Выключить лимиты`

При превышении лимита:
`🚨 Лимит «{category}» превышен на {delta}. Что делаем?`

Кнопки:

- `Оставить как есть`
- `Перенести в "Другое"`

## 11. История и правки

Экран `🧾 История`:
- список последних операций
- кнопки: `1`, `2`, `3`, `Показать ещё`

После выбора операции:
- `✏️ Изменить сумму`
- `🔁 Изменить категорию`
- `🗑 Удалить`
- `Назад`

## 12. Настройки
- Валюта
- Период по умолчанию (`месяц`/`неделя`)
- День начала месяца (зарплатный цикл)
- Напоминания (`ежедневно`, `по будням`, `раз в неделю`, `выкл`)
- Экспорт (`CSV`, `Google Sheets`)

## 13. Анти-фейлы и валидация
- Если пришло только число: спросить тип операции и категорию.
- Нормализация суммы:
  - `25 000`, `25000р`, `25.000,50` -> в число.
- `<= 0`: отклонять и просить корректную сумму.
- Пустая категория расхода: просить выбрать.
- Смешанная валюта:
  - MVP: запретить и просить переключить валюту в настройках.
  - Advanced: включить авто-конвертацию.
- Если лимиты выключены:
  - не отправлять тревожные сообщения, только долю категории в расходах.

## 14. Регулярные выражения для быстрого парсинга
- Расход (`категория сумма`):
  - `^(?<cat>[\\p{L}\\-\\s]{2,32})\\s+(?<amount>\\d[\\d\\s.,]*\\d|\\d)$`
- Расход (`сумма категория`):
  - `^(?<amount>\\d[\\d\\s.,]*\\d|\\d)\\s+(?<cat>[\\p{L}\\-\\s]{2,32})$`
- Доход:
  - `^(зп|зарплата|доход)\\s+(?<amount>\\d[\\d\\s.,]*\\d|\\d)$`
- Лимит:
  - `^лимит\\s+(?<cat>[\\p{L}\\-\\s]{2,32})\\s+(?<amount>\\d[\\d\\s.,]*\\d|\\d)$`

## 16. Тон сообщений
- Коротко, дружелюбно, без длинных текстов.
- Всегда завершать действие явным подтверждением `✅`.
- После ключевых действий показывать 1-2 полезные метрики (баланс, лимит категории).

## 17. Критерии готовности MVP
1. Пользователь может добавить расход кнопкой и быстрым текстом.
2. Пользователь может добавить доход кнопкой и быстрым текстом.
3. Работают итоги за текущий месяц.
4. Работают лимиты по категориям с предупреждением 80% и превышением.
5. Работают просмотр истории и базовые правки.
6. Есть настройки валюты и периода.
